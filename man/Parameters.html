PhosFake Pipeline Parameters

This document provides a detailed description of the parameters used in the PhosFake pipeline. PhosFake is a tool for generating an in-silico bottom-up phosphoproteomics dataset. The pipeline simulates various stages of the experimental process, from the generation of ground truth data to mass spectrometry (MS) analysis and statistical testing. For more information about the project, visit PhosFake on GitHub.

Experimental Design

Param$NumCond: The number of different experimental conditions to be tested. For example, if you are comparing treated versus untreated samples, this value would be 2. This parameter is crucial for setting up the experimental design. Currently set to 2. Number of Replicates

Param$NumReps: The number of biological or technical replicates for each condition. Replicates are essential for statistical validity and reproducibility of the results. Ensuring adequate replication helps in reducing variability and increasing the reliability of the results. Currently set to 5. Input Data Paths Path to FASTA File

Param$PathToFasta: The file path to the FASTA file containing the protein sequences. This file is essential for identifying and quantifying proteins in your samples. Accurate protein identification depends heavily on the quality and completeness of the FASTA file. Currently set to InputData/fasta_full_human.fasta. Path to Protein List

Param$PathToProteinList: The file path to a list of proteins of interest, if any. This can be used to focus the analysis on a subset of proteins. Providing a specific list can streamline the analysis and enhance focus on target proteins. Currently set to NULL. Ground Truth Proteoform-Centric Data Generation Proteoform IDs Generation

Param$FracModProt: The fraction of proteins selected to be modified. This parameter controls how many proteins will undergo post-translational modifications (PTMs). It helps in simulating realistic biological scenarios. Set to 0.0, indicating no proteins will be modified.

Param$FracModPerProt: The fraction of modifiable proteins sampled for modifications. A value of 2 means twice as many proteoforms as the selected proteins will be generated. This ensures diversity in the generated proteoforms.

Param$PTMTypes: The types of PTMs to be considered. Currently set to ph for phosphorylation. This parameter allows for the inclusion of different types of PTMs in the simulation.

Param$PTMTypesDist: The distribution of PTM types. Currently set to 1, indicating 100% phosphorylation. This controls the relative frequency of each PTM type.

Param$PTMTypesMass: The mass shifts associated with the PTM types. Currently set to 79.9663 for phosphorylation. Accurate mass shifts are crucial for the identification and quantification of PTMs.

Param$PTMMultipleLambda: The lambda parameter for the Poisson distribution of multiply modified proteins. Set to 0.5, scaled to the number of possible PTM sites. This parameter models the frequency of multiple modifications on the same protein.

Param$ModifiableResidues: The residues where PTMs can occur. For phosphorylation, these are serine (S), threonine (T), and tyrosine (Y). This parameter defines the possible modification sites on the proteins.

Param$ModifiableResiduesDistr: The relative distribution of modifiable residues. Set to 0.86 for serine, 0.13 for threonine, and 0.01 for tyrosine. This reflects the observed frequencies of modifications at these residues in real biological data.

Param$RemoveNonModFormFrac: The fraction of modifiable proteins without non-modified forms. Set to 0. This parameter controls the presence of non-modified forms of proteins in the simulation. Proteoform Quantities Generation

Param$QuantColnames: The column names for quantitative values, structured as C_R. This provides a clear and organized structure for the quantitative data.

Param$QuantNoise: The general noise level for all quantitative values, given as the standard deviation of a normal distribution. Set to 0.0. This parameter simulates measurement noise in the quantitative data.

Param$DiffRegFrac: The fraction of differentially regulated proteoforms. Set to 0.0. This parameter allows for the simulation of differential regulation, mimicking biological variability.

Param$DiffRegMax: The maximum amplitude of difference for differentially regulated proteins on a log2 scale. Set to 3. This parameter defines the range of fold changes for differentially regulated proteins.

Param$UserInputFoldChanges: User-defined fold changes for regulated proteoforms. Set to NULL. This allows for custom input of fold changes if specific regulation patterns are desired.

Param$ThreshNAProteoform: The threshold for removing quantitative values at the proteoform level. Set to -100. This parameter filters out low-quality data points based on intensity thresholds.

Param$AbsoluteQuanMean: The mean value for absolute quantitation. Set to 30.5. This parameter sets the baseline for absolute quantitation values.

Param$AbsoluteQuanSD: The standard deviation for absolute quantitation. Set to 3.6. This parameter models the variability in absolute quantitation.

Param$ThreshNAQuantileProt: The quantile threshold for removing quantitative values. Set to 0.01. This parameter ensures that only reliable quantitative values are retained. Enzymatic Digestion Parameters Enzyme

Param$Enzyme: The enzyme used for protein digestion. Currently set to trypsin. Trypsin is commonly used in proteomics for its specificity and efficiency in cleaving proteins. Proportion of Peptides with Missed Cleavages

Param$PropMissedCleavages: The proportion of peptides that have missed cleavages. Set to 1. Missed cleavages occur when the enzyme does not cleave at all possible sites, which can affect peptide identification. Maximum Number of Missed Cleavages

Param$MaxNumMissedCleavages: The maximum number of missed cleavages per peptide. Set to 6. This parameter defines the upper limit of missed cleavages allowed in the simulation. Peptide Length Filter

Param$PepMinLength: The minimum length of peptides to be considered. Set to 7. This filter ensures that only peptides of a certain length are included, which is important for reliable identification.

Param$PepMaxLength: The maximum length of peptides to be considered. Set to 30. This parameter excludes excessively long peptides that might not be efficiently analyzed by MS. Parallel Computing

Param$Cores: The number of CPU cores to use for parallel computing. Set to 1. Using multiple cores can significantly speed up the computation.

Param$ClusterType: The type of cluster to use for parallel computing. Currently set to PSOCK. The cluster type determines how parallel processing is managed. Least Abundant Peptide Loss

Param$LeastAbundantLoss: The percentage of least abundant peptides to be removed. Set to 0.0. This parameter allows for the exclusion of peptides that are too low in abundance to be reliably quantified. Sample Preparation Simulation Parameters Enrichment Loss

Param$EnrichmentLoss: The loss of phosphorylated peptides during enrichment. Set to 0.2. This parameter models the inefficiencies in the enrichment process, where some peptides are lost. Enrichment Efficiency

Param$EnrichmentEfficiency: The efficiency of enrichment for phosphorylated peptides. Set to 1. This parameter reflects how effectively phosphorylated peptides are enriched relative to non-modified peptides. Non-Modified Peptide Signal Loss

Param$EnrichmentNonModSignalLoss: The loss of signal for non-modified peptides during enrichment. Set to 0. This parameter controls the reduction in signal for non-modified peptides that contaminate the enriched fraction. Enrichment Protocol Noise

Param$EnrichmentNoise: The noise introduced by the enrichment protocol. Set to 0.2. This parameter adds variability to the enrichment process, simulating real experimental conditions
